# TagLock – GitHub Pages Starter

Dies ist ein minimaler, schicker One‑Pager für das TagLock‑Projekt. Er lässt sich direkt über GitHub Pages deployen.

## Schnellstart
1. Neues Repository erstellen, z.B. `taglock-site`.
2. Diese Dateien ins Repo legen (Root).
3. In den **Repository Settings › Pages** als Source **GitHub Actions** wählen.
4. Der mitgelieferte Workflow deployt bei jedem Push auf `main` automatisch.

Die Platzhalter `USER_OR_ORG/REPO` in `index.html` kannst du auf euer echtes Repo ändern.
